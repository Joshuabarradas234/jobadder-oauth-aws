Step 1 of 5
Jobs queued in SQS
5 candidate fetch requests have been added to the AWS SQS queue. Each one is independent — if any fails, it retries automatically up to 3 times before alerting your team.
No recruiter action needed. The queue handles volume, retries and ordering automatically — at any scale.
Step 2 of 5
Secure endpoint receives the request
AWS API Gateway is your secure front door — always on, rate-limited, and protected. It routes the request to the correct Lambda function in milliseconds.
Available 24/7. No server to manage. Scales to thousands of concurrent requests without configuration.
Step 3 of 5
Credentials retrieved securely
The OAuth token is retrieved from AWS Secrets Manager and decrypted via KMS — the same encryption standard used by banks. No credentials are ever stored in code, logs or environment variables.
Zero credential exposure. Passes ISO 27001:2022 and SOC 2 audit requirements out of the box.
Step 4 of 5
Fetching candidate data from JobAdder
The Fetcher Lambda is calling JobAdder's API for each candidate simultaneously. Your recruiters are doing nothing — this runs entirely in the background while they focus on placements.
Parallel processing. Multiple candidates are fetched at the same time — volume doesn't slow it down.
